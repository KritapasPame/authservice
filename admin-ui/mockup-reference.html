สรุป API ทุกเส้นของ 3 หน้านี้

Endpointใช้ในหน้าสถานะหมายเหตุ

POST /signup/personal2ใหม่public — สมัคร Personal เอง (Zitadel register + สร้าง tenant type=personal)

สิ่งที่ DB ยังไม่มี (ต้องเพิ่มก่อนต่อ API จริง)

  • user_permissions (user_id, company_id, permission_id) — source of truth สิทธิ์รายคน
  • users.is_group_admin + user_companies.is_admin — role ตายตัว 2 ตัวฝั่งลูกค้า
  • user_companies.position — ป้ายตำแหน่ง (แสดงผลอย่างเดียว)
  • presets + preset_permissions — แทน roles/role_permissions เดิม
  • packages (id, name, seat_limit, company_limit, admin_limit, doc_limit_monthly, allow_group_admin, self_signup) + package_permissions — นิยามแพ็กเกจ + function ที่ใช้ได้
  • tenants.package_id + tenants.type ('org' | 'personal') — เจ้าไหนซื้อแพ็กไหน / เป็นองค์กรหรือบุคคล
  • invoices (id, tenant_id, number, description, amount, status, issued_at, paid_at) — บิล/ใบเสร็จ (พิมพ์จากหน้า HTML)
