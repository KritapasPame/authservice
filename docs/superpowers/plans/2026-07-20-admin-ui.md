# Admin UI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax.

**Goal:** หน้า admin จริง 4 หน้า ตาม mockup (ลูกค้า/แพ็กเกจ/รายละเอียด/ตั้งสิทธิ์ราย user) ต่อ API entitlement ที่ merge แล้ว + login ผ่าน Zitadel PKCE

**Architecture:** vanilla HTML/JS/CSS ไม่มี framework ไม่มี dependency ใหม่ — ไฟล์ static อยู่ `admin-ui/` served โดย entitlement service เอง (same-origin ไม่ต้อง CORS) เส้น `/admin` ; JS แยกไฟล์ต่อหน้า + `api.js` (fetch wrapper) + `auth.js` (OIDC PKCE, token ใน sessionStorage)

**Tech Stack:** Bun (มีอยู่แล้ว), Elysia static route, browser crypto.subtle (PKCE)

## Global Constraints

- โค้ดง่ายสุด one-liner ได้ทำ — vanilla JS ES modules ตรง ๆ ไม่มี build step (browser โหลด .js ตรง)
- หน้าตา/CSS: ยกจาก mockup reference (`admin-ui/mockup-reference.html` — T1 copy เข้า repo) ใช้ token/class เดิมทั้งหมด ภาษาไทยเหมือนเดิม
- ทุก fetch ผ่าน `api.js` เท่านั้น: แนบ `Authorization: Bearer <token>` จาก `auth.js`; 401 → เด้ง login; error อื่นโชว์ toast ข้อความจาก body
- endpoint ตรงตาม spec V2 (`docs/superpowers/specs/2026-07-19-per-user-permissions-packages-design.md` — ดู §API + §Implementation notes สำหรับ path จริง เช่น `GET /users/tenant/:tenantId`)
- ทดสอบ: logic ล้วน (PKCE helper, url builder) ใช้ `bun test` ใน `admin-ui/tests/`; ส่วน UI = manual checklist ใน `admin-ui/README.md` (API มี test ครบแล้วฝั่ง entitlement)
- commit ลงท้าย `Co-Authored-By: Claude Opus 4.8 (1M context) <noreply@anthropic.com>`
- Wave: T1 → (T2, T3, T4 ขนาน — คนละไฟล์หน้า) → T5

### Task 1: Scaffold + auth + static serving

**Files:** สร้างตาม File Structure แถว T1 ทั้งหมด; Modify `entitlement/src/http/app.ts`

**Interfaces (Produces):**
- `auth.js`: `login()` (redirect authorize+PKCE), `handleCallback()` (แลก code→token เก็บ sessionStorage), `getToken()`, `claims()` (decode JWT payload, ไม่ verify — server verify อยู่แล้ว), `isSuperadmin()`, `logout()`
- `api.js`: `api.get(path)`, `api.post(path, body)`, `api.put`, `api.patch`, `api.del` → คืน json; โยน toast เมื่อ error; 401 → `login()`
- `router.js`: `route(pattern, renderFn)`, `navigate(hash)`, render เข้า `<main id="view">`; nav item ซ่อนตาม `isSuperadmin()`
- static.ts: `GET /admin` → index.html, `GET /admin/*` → ไฟล์ใน admin-ui/ (Bun.file, 404 ถ้าไม่มี; กัน path traversal ด้วย normalize + startsWith check)

- [ ] Step 1: copy mockup (หา html ใน scratchpad หรือใช้โครงจาก artifact — controller จะแนบ path ให้) → `admin-ui/mockup-reference.html`; ดึง `<style>` ไป `styles.css`
- [ ] Step 2: เขียน auth.js — PKCE: `verifier = base64url(crypto.getRandomValues(32B))`, `challenge = base64url(sha256(verifier))`, authorize url `${issuer}/oauth/v2/authorize?client_id&redirect_uri=${origin}/admin&response_type=code&scope=openid profile urn:zitadel:iam:org:projects:roles&code_challenge&code_challenge_method=S256`; callback แลก token ที่ `${issuer}/oauth/v2/token` (form-encoded) เก็บ access_token+id ใน sessionStorage
- [ ] Step 3: api.js + router.js + index.html (sidebar: ลูกค้า/แพ็กเกจ [superadmin], ตั้งสิทธิ์ผู้ใช้ [ทุก admin], ปุ่ม logout) + toast component
- [ ] Step 4: entitlement static.ts + mount + test (GET /admin → 200 text/html; /admin/../secrets → 404)
- [ ] Step 5: `bun test admin-ui/tests/ && cd entitlement && bun test tests/static.test.ts` เขียว → commit

### Task 4: หน้า 4 ตั้งสิทธิ์ราย user (groupcompanyadmin/admin)

**Files:** `admin-ui/src/pages/permissions.js`

**Interfaces:** tenantId มาจาก `claims()['urn:platform:tenantId']` (superadmin: ใส่ช่องเลือก tenant จาก GET /tenants) — `GET /users/tenant/:tenantId`, `GET /companies/:tenantId`, `GET /users/:id/permissions?companyId=`, `PUT /users/:id/permissions {companyId, position, permissionKeys}`, `PATCH /users/:id/admin`, `GET /presets/:tenantId`, `POST /presets`, `POST /users/invite {tenantId,email,companyIds,presetSlug?|permissionKeys?}`, `PATCH /users/:id/status`
